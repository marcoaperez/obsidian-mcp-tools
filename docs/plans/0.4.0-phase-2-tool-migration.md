# 0.4.0 — Phase 2: Tool handler migration

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking. Tasks 3-21 (tool ports) are independent and parallelizable in batches of 3-5 once the setup phase (T1-T2) completes.

**Goal:** Bring `feat/http-embedded` to feature parity with `0.3.7` by porting all 19 remaining MCP tools from `packages/mcp-server/src/features/*/index.ts` into the in-process plugin (`packages/obsidian-plugin/src/features/mcp-tools/tools/`). Each tool stops calling Local REST API via `makeRequest` and instead uses the native Obsidian API (`app.vault.*`, `app.workspace.*`, `app.metadataCache.*`, `app.fileManager.*`, `app.commands.*`, `requestUrl`).

**Architecture:** One file per tool in `tools/`, exporting `<toolName>Schema` (ArkType) + `<toolName>Handler` (async function taking a context object). The context contains references to `app`, the plugin instance (for accessing other plugins' APIs and the rate limiter), and the bearer token for clients that need to relay it. `mcp-tools/index.ts` orchestrates registration. `command-permissions/`, `tool-toggle/`, and the rate limiter are reused intact from the existing plugin codebase.

**Tech Stack:** TypeScript strict + `verbatimModuleSyntax`, ArkType 2.x, Bun test runner, Obsidian Plugin API (`obsidian` module — value imports OK in `obsidian-plugin` package), `@modelcontextprotocol/sdk@1.29.0`, optional dependencies on Smart Connections and Templater plugins (loaded reactively via existing `loadSmartSearchAPI`/`loadTemplaterAPI` helpers).

**Design reference:** `docs/design/2026-04-24-http-embedded-design.md` § Tool implementation. Decisions referenced by ID (D1-D9) where relevant.

**Branch:** `feat/http-embedded`. Latest commit at the start of Phase 2: `6afc31c` (merge of main).

```typescript
// getActiveFile.ts
import { type } from "arktype";
import type { App } from "obsidian";

export const getActiveFileSchema = type({
  name: '"get_active_file"',
  arguments: {
    "format?": '"markdown"|"json"',
  },
}).describe(
  "Returns content of the currently active note. Default format is markdown; pass format=json to receive an object with content, frontmatter, tags, and path.",
);

export type GetActiveFileContext = {
  arguments: { format?: "markdown" | "json" };
  app: App;
};

export async function getActiveFileHandler(
  ctx: GetActiveFileContext,
): Promise<{ content: Array<{ type: "text"; text: string }>; isError?: boolean }> {
  const file = ctx.app.workspace.getActiveFile();
  if (!file) {
    return {
      content: [{ type: "text", text: "No active file." }],
      isError: true,
    };
  }

  const content = await ctx.app.vault.read(file);
  if (ctx.arguments.format !== "json") {
    return { content: [{ type: "text", text: content }] };
  }

  const cache = ctx.app.metadataCache.getFileCache(file);
  const frontmatter = (cache?.frontmatter as Record<string, unknown>) ?? {};
  const tags = Array.isArray(frontmatter.tags) ? frontmatter.tags : [];

  const body = {
    path: file.path,
    content,
    frontmatter,
    tags,
  };
  return { content: [{ type: "text", text: JSON.stringify(body, null, 2) }] };
}
```
